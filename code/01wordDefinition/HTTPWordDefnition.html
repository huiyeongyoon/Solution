1. HTTP: 클라이언트에서 서버까지 이련의 흐름을 결정하는것 2. 프로토콜: 서로 다른 하드웨어와 운영체제 등을 가지고 서로 통신 하는 모든 요소에 규칙 3. TCP/IP: 인터넷 관련된 프로토콜집합 1) 애플리케이션 계층: 유저에게 제공되는 애플리케이션에서 사용하는 통신의 움직임을 결정 2) 트랜스포트 계층: 애프리케이션 계층에 네트워크로 접속되어 있는 2대의 컴퓨터 사이의 데이터 흐름 제공 TCP, UDP 3) 네트워크 계층: 네트워크 상에서 패킷의 이동 (어떠한경로로,어디에 보낼지) 4) 링크 계층: 네트워크에 접속하는 하드웨어적 단면를 다룸 왜계층으로? 1) 사양변경시 전체 말고 해당 계층만 변경하면되서 2) 계층별 내부를 자유롭게 편하게 설계 가능하여 4. 패킷: 전송하는 데이터의 최소 단위 5.송신시: 애플리케이션 계층 -> 링크계층 6.수신시: 링크계층-> 애플리케이션 계층 7.url 포맷 1) 스키마: http, https 같은 스키마를 사용하여 리소스를 얻기위하여 프로토콜을 지시한다 2) 크리덴셜: 서버로부터 리소를 취득하기위한 것 ex:유저명 패스워드 3) 서버주소: www.hacker.jp와 같은dns 이름이나 192,168,1,1 같은 IPv4 주소 4) 서버포트: 포트번호 5) 계층적 파일패스: 특정 리소스를 식별하기위한 서버상의 파일 패스 6) 쿼리문자열: 파일패스로 지정된 임의의 파라미터를 넘겨주기 위해 쿼리 문자열을 사용한다 7) 프래그멘트 식별자: 취득한 리소스에서 서브 리소스를 가르키기 위해서 프래그멘트 식별자가 사용된다 8.클라이언트: 텍스트 이미지등과 같은 리소스를 요구하는쪽 9.서버: 리소스를 제공하는 쪽
